## Part-1: Software Testing Fundamentals

### 1. What is Testing?
Software testing is a set of activities to discover defects and evaluate the quality of software.  
It checks whether the software satisfies business requirements or not.

### 2. Why Software Testing is Necessary?
- Bug Detection  
- Quality Assurance  
- Customer Satisfaction  
- Cost Savings  
- Risk Mitigation  
- Security  
- Performance Optimization  
- Compliance  
- Better User Experience  
- Business Growth

 ### 3. Testing Objectives
- Functional correctness  
- Authorization & Security  
- Service level (Load, Performance, Responsiveness)  
- Usability  
- Verify requirements, reduce risk, build confidence

### 4. Quality VS Quality Assurance 
- **Quality**: Degree of excellence of a product/service
- **Quality Assuarance**: Systematic process to ensure the product meets specified requirements

### 5. Software Testing Misconceptions (Reality Check)
- Testing is not just “clicking around”
- esting is not easy, needs skills and knowledge
- 100% bug-free software is impossible
- Testing is more than finding bugs
- Automation cannot fully replace manual testing
- Quality is everyone’s responsibility (not only testers)
- Testing does not delay release — it prevents bigger delays later

### 6. Verification VS Validation
------------------------------------------------------------------------------------
| Aspect          | Verification                  | Validation                     |
|-----------------|-------------------------------|--------------------------------|
| Purpose         | Right process & documents     | Right product for business     |
| Timing          | Early (before coding)         | Later (after coding)           |
| Type            | Static testing                | Dynamic testing                |
| Part of         | QA                            | Software Testing               |
------------------------------------------------------------------------------------

### 7. Quality Assurance VS Quality Control
--------------------------------------------------------------------------------------------
| Aspect              | QA (Quality Assurance)          | QC (Quality Control)             |
|---------------------|---------------------------------|----------------------------------|
| Focus               | Prevent defects                 | Detect & fix defects             |
| Approach            | Preventive                      | Corrective                       |
| When                | Before execution                | After execution                  |
| Process             | Verification                    | Validation                       |
--------------------------------------------------------------------------------------------

### 8. Testing VS Debugging 
- **Testing** -> Find out the failures
- **Debugging** -> Find out the root cause and fixes it

### 9. Error, Defect, Bug & Failure
- **Error**: Human mistake
- **Defect**: Problem in code/documentation
- **Bug**: Defect accepted by developer
- **Failure**: When user sees the problem

### 10. Root Cause 
Main reason behind the defect. Fixing root cause prevents same issues in future.

### 11. 7-Testing Principles 
1. Testing shows presence of defects (not absence)
2. Exhaustive testing is impossible
3. Early testing saves time & money
4. Defects cluster together (Pareto principle)
5. Tests wear out (Pesticide Paradox)
6. Absence-of-defects fallacy
7. Testing is context dependent
